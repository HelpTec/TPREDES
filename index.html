01 — Virtualización de Red

¿Qué es una VLAN?

Definición

Una VLAN es una red lógica que agrupa dispositivos conectados físicamente a una misma red local, segmentándolos para formar redes independientes. Mejora la seguridad, gestiona el tráfico de forma eficiente y simplifica la administración.

🔒

Seguridad

Aísla departamentos y tipos de tráfico, evitando que se interfieran entre sí.

⚡

Eficiencia

Gestiona el tráfico de forma más eficiente mejorando el rendimiento general de la red.

🏷️

VLAN ID

Opera en la Capa 2 del modelo OSI etiquetando tramas con identificadores únicos llamados VLAN IDs.

02 — Privacidad

¿Qué es una VPN?

Una VPN establece una conexión de red privada entre dispositivos a través de Internet, permitiendo transmitir datos de forma segura y anónima sobre redes públicas. Enmascara las IPs y cifra los datos haciéndolos ilegibles para terceros.

👁️

Privacidad

Oculta la actividad del usuario y su dirección IP real ante terceros en la red.

🕵️

Anonimato

Enmascara la identidad digital del usuario durante la navegación y transferencia de datos.

🛡️

Seguridad

Cifra los datos durante la transmisión, protegiéndolos de accesos no autorizados.

03 — Almacenamiento

¿Qué es una SAN?

Storage Area Network

Red de alta velocidad que conecta servidores con dispositivos de almacenamiento. Centraliza el almacenamiento en bloques permitiendo a las organizaciones almacenar datos en un pool compartido.

LUN — Logical Unit Numbers

Los datos se almacenan en unidades lógicas (LUN). Los servidores los ven como discos locales: los particionan, formatean y usan para almacenar datos.

05 — Comunicación

Protocolos de Comunicación

¿Qué es un protocolo?

Conjunto de reglas, normas y procedimientos estandarizados que permiten a dos o más dispositivos conectarse, intercambiar información y entenderse en una red. Definen el formato de mensajes y gestionan la detección de errores.

TCP/IP

Protocolo fundamental de Internet para transmisión de paquetes.

HTTP/S

Transferencia de información en sitios web (S = cifrado).

FTP

Orientado a la transferencia de archivos entre sistemas.

SMTP

Utilizado para el envío de correos electrónicos.

06 — Comparativa

TCP/IP vs NetBIOS

TCP/IP

Conjunto completo de protocolos para comunicación global incluyendo Internet. Usa direcciones IP. Permite comunicación entre redes diferentes. Es el estándar universal actual organizado en capas: acceso a red, internet, transporte y aplicación.

NetBIOS

Interfaz orientada a redes locales. Usa nombres de red para identificar dispositivos. No es un protocolo completo por sí mismo, depende de otros (ej: NetBT sobre TCP/IP). Fue ampliamente usado en redes Windows antiguas. Actualmente en desuso.

10 — Configuración Dinámica

Servicio DHCP

Dynamic Host Configuration Protocol

Asigna automáticamente direcciones IP a cada dispositivo de red (host). Elimina la necesidad de asignaciones manuales por parte de administradores. También asigna: máscaras de subred, puertas de enlace y servidores DNS.

11 — Resolución de Nombres

Servicio DNS

El DNS traduce nombres de host a direcciones IP, funcionando como una agenda telefónica de Internet. Por ejemplo: www.apple.com → 17.253.207.54

🔄

Servidor DNS Recursivo

Recibe la consulta del cliente y se encarga de buscar la respuesta consultando a otros servidores DNS hasta encontrarla.

🏛️

Servidor DNS Autoritativo

Contiene la información definitiva sobre un dominio específico. Es la fuente final de verdad para la resolución del nombre.

// Flujo de resolución DNS
1.Usuario escribe www.ejemplo.com en el navegador
2.El SO consulta al Servidor DNS Recursivo
3.El recursivo consulta al Servidor DNS Autoritativo
4.Responde con la IP: 93.184.216.34
5.El navegador se conecta directamente a esa IP
12 — Comunicaciones Inalámbricas

Tecnologías Wireless

Se denomina wireless a las tecnologías que transmiten información sin un medio físico guiado, mediante radiofrecuencia. Difieren en alcance, potencia, ancho de banda, latencia y consumo energético.

📶

Wi-Fi / WLAN — IEEE 802.11

La más difundida en hogares, oficinas y campus. Las generaciones se nombran comercialmente:

Wi-Fi 4 = 802.11n Wi-Fi 5 = 802.11ac Wi-Fi 6 = 802.11ax

Wi-Fi 6 fue diseñado especialmente para entornos densos con múltiples usuarios.

🎧

Bluetooth / WPAN

Diseñado para corto alcance y consumo contenido. Ideal para periféricos, audio, wearables y enlaces entre dispositivos personales. Administrado por Bluetooth SIG (históricamente vinculado a IEEE 802.15.1).

🏠

Zigbee / IoT — IEEE 802.15.4

Muy usado en automatización, sensores y domótica. Privilegia bajo consumo y redes de baja tasa de datos. Permite dispositivos con larga autonomía y topologías tipo malla.

📡

Redes Móviles: 4G / 5G

Regidas por el ecosistema 3GPP. Define interfaz de radio, arquitectura de red, movilidad y autenticación. 5G es un sistema completo de telecomunicaciones, no solo "radio más rápida".

Ejemplo en una universidad

Las notebooks usan Wi-Fi 6 dentro del campus · Los auriculares del personal usan Bluetooth · Los sensores de temperatura en aulas usan Zigbee · Los usuarios fuera del edificio usan 4G/5G del operador.

13 — Intermediación de Red

¿Qué es un Proxy?

Un proxy es un intermediario que recibe solicitudes de un nodo y las reenvía hacia otro. Su valor técnico está en que termina o intermedia la comunicación, permitiendo aplicar políticas, autenticación, caché, anonimización y filtrado.

→

Forward Proxy

Se ubica del lado del cliente. Los usuarios de una red interna no salen directamente a Internet sino a través del proxy. Permite control, registro, filtrado de navegación y caché de contenido.

←

Reverse Proxy

Se ubica delante de uno o varios servidores. Los clientes externos creen acceder a un único servicio, pero el proxy reenvía internamente al backend. Permite ocultar servidores, repartir carga y centralizar TLS.

14 — Prevención de Bucles

Protocolo Spanning Tree

IEEE 802.1D — Capa 2 OSI

STP evita bucles lógicos en redes Ethernet con enlaces redundantes. Sin él, una trama broadcast puede circular indefinidamente generando tormentas de broadcast y degradación severa.

Funcionamiento

Los switches usan BPDUs para elegir un root bridge, calcular el mejor camino hacia él y dejar algunos puertos en estado de bloqueo. La topología física mantiene redundancia, pero la lógica queda libre de bucles.

STP — 802.1D RSTP — más rápido MSTP — escalable

STP bloquea el enlace redundante para evitar el loop

15 — Protocolo de Enrutamiento

Protocolo OSPF

Open Shortest Path First — RFC 2328 — Capa 3

Protocolo de enrutamiento interior (IGP) de tipo link-state. Se usa dentro de un mismo sistema autónomo para que los routers compartan información de topología y calculen rutas mediante el algoritmo SPF (Dijkstra).

Link-State
Cada router anuncia el estado de sus enlaces y todos construyen una base de datos topológica completa.
Algoritmo SPF
Calcula el mejor camino según un costo. Permite decisiones más finas que simplemente contar saltos.
Áreas OSPF
Admite segmentación en áreas mejorando escalabilidad en redes medianas o grandes.

vs STP

STP opera entre switches en capa 2 para prevenir loops. OSPF opera entre routers en capa 3 para calcular rutas óptimas. Son protocolos complementarios, no equivalentes.

16 — Resolución de Direcciones

Protocolo ARP

ARP (Address Resolution Protocol — RFC 826) permite asociar una dirección lógica IPv4 con una dirección física MAC en Ethernet. Opera dentro del dominio de enlace local, no sirve para encontrar hosts en Internet.

❓

ARP Request (broadcast)

Cuando un host necesita la MAC de una IP destino y no la tiene en caché, envía un broadcast preguntando: "¿Quién tiene esta IP?"

✅

ARP Reply (unicast)

El equipo dueño de esa IP responde con su MAC. La relación IP↔MAC se guarda en caché temporalmente para reutilizarla.

Relación con DNS y Gateway

DNS responde "qué IP tiene ese nombre". ARP responde "qué MAC tiene el siguiente salto". Si la IP destino no está en la subred local, ARP resuelve la MAC del gateway predeterminado, no del servidor remoto.

17 — Seguridad de Red

¿Qué es un Firewall?

Un firewall filtra, inspecciona y controla el tráfico entrante y saliente según reglas definidas. Actúa como barrera de control entre zonas con distinto nivel de confianza. Puede ser hardware, software en un host, o un servicio administrado.

IP Origen/Destino
Permite o bloquea según la dirección IP de origen y destino del paquete.
Puerto / Protocolo
Filtra por puerto origen/destino y protocolo (TCP, UDP, ICMP…).
Estado de Conexión
Los firewalls stateful inspeccionan el estado de la conexión, no solo el paquete individual.
18 — Zona Perimetral

¿Qué es una DMZ?

Demilitarized Zone

Red intermedia donde se colocan sistemas que deben recibir conexiones desde redes menos confiables (Internet) sin exponer directamente la red interna. También llamada screened subnet.

Reglas de Firewall en DMZ

Deben existir reglas muy específicas entre: Internet ↔ DMZ · DMZ ↔ Red interna · Red interna ↔ DMZ. Si un sistema en la DMZ es comprometido, el atacante no debería tener paso libre hacia la LAN interna.

19 — Punto de Salida

¿Qué es un Gateway?

Default Gateway

En el contexto IP, el gateway es el punto de salida hacia otra red. Normalmente es un router o interfaz de router al que los hosts envían tráfico destinado a redes remotas. Un host compara la IP destino con su subred: si el destino no pertenece a la misma red, envía el paquete al gateway predeterminado.

// Ejemplo

Una PC con IP 192.168.1.50/24 quiere llegar a 8.8.8.8. Como esa IP no pertenece a 192.168.1.0/24, no intenta buscarlo directamente. En cambio, resuelve por ARP la MAC del gateway 192.168.1.1 y le entrega el paquete para que lo enrute mediante OSPF u otras rutas.

21 — Conectividad WAN

Tipos de Enlace

🏷️

MPLS

Tecnología de transporte usada por operadores para enrutar tráfico mediante labels. Ofrece control de calidad de servicio, rutas predecibles y acuerdos de servicio estrictos. Ideal para empresas con muchas sedes y tráfico crítico.

🔒

LAN to LAN (Site-to-Site VPN)

Enlace lógico entre dos redes LAN mediante túneles VPN cifrados sobre Internet. Los hosts de una LAN se comunican con la otra sin exponer tráfico en claro. Bajo costo, sin contratar red privada del carrier.

📡

Microonda

Enlace inalámbrico punto a punto mediante radiofrecuencia terrestre. Requiere línea de vista entre antenas. Baja latencia y buena capacidad. Ideal para unir dos edificios o campus donde tender fibra es costoso.

🛰️

VSAT

Enlace satelital mediante terminales de pequeña apertura. Su gran ventaja es la cobertura geográfica extrema: llega donde los medios terrestres no llegan. Usado en sitios rurales, minas, pozos y operaciones temporales.

💡

Fibra Dedicada / EPL

Enlace privado punto a punto con ancho de banda reservado, baja latencia y alta previsibilidad. El proveedor entrega una conexión de capa 2 con SLA estrictos. Ideal para interconexión sede ↔ data center.

📱

LTE/4G Fijo

Enlace WAN sobre red celular LTE. Muy rápido de desplegar si hay cobertura. Útil para sucursales pequeñas, enlaces de respaldo o sitios temporales sin infraestructura fija.

22 — Acceso Móvil

Tecnología LTE

LTE (Long Term Evolution) es una tecnología de acceso móvil del ecosistema 3GPP, evolución de UMTS/HSPA. Orientada a mayor capacidad, menor latencia, mejor eficiencia espectral y arquitectura basada en IP.

📱

UE

User Equipment — el dispositivo del usuario que se conecta a la red LTE.

📶

E-UTRAN

Red de acceso radio formada por estaciones base llamadas eNodeB.

☁️

EPC

Evolved Packet Core — núcleo de red orientado a paquetes IP.

OFDMA (downlink) SC-FDMA (uplink) MIMO Menor latencia que 3G Eficiencia espectral mejorada Arquitectura IP simplificada
23 & 24 — Colaboración y Calidad

Microsoft Teams & QoS en MPLS

💬

Microsoft Teams

Plataforma de comunicación y colaboración basada en la nube. Usa TCP/IP para transmisión de datos y streaming para videollamadas. Optimiza el ancho de banda mediante compresión de audio/video y puede integrarse con políticas de QoS para priorizar tráfico en redes corporativas con MPLS. Alternativas: Zoom, Slack.

⚖️

QoS en MPLS

Aplicar calidad de servicio en MPLS significa priorizar tráfico crítico (voz, video) sobre otros datos. Se logra mediante clasificación y marcado del tráfico con valores DSCP. MPLS facilita este proceso con su enrutamiento eficiente por etiquetas, mientras que QoS gestiona la prioridad para evitar congestiones y garantizar el rendimiento de aplicaciones sensibles al retardo.

26 — Certificaciones

Certificaciones Cisco

CCENT

Nivel inicial. Valida conocimientos básicos: conectividad, direccionamiento IP y configuración simple de dispositivos.

CCNA

Nivel intermedio. Instalar, configurar y administrar redes pequeñas y medianas: routing, switching, seguridad básica y troubleshooting.

CCNP

Nivel avanzado. Diseñar, implementar y gestionar redes complejas con mayor profundidad en protocolos y soluciones empresariales.

Track: Routing & Switching

Diseño, configuración y administración de redes cableadas. Incluye routers, switches, protocolos de enrutamiento (OSPF, EIGRP) y segmentación de redes.

Track: Security

Protección de redes: firewalls, VPNs, control de accesos y detección de amenazas. Garantiza integridad y confidencialidad de la información.

28 & 29 — Estándares IEEE

Estándares IEEE 802

IEEE 802.3 — Ethernet

Define el funcionamiento de redes Ethernet cableadas. Se implementa con cables UTP o fibra óptica y utiliza un modelo de comunicación basado en tramas.

✓ Bajo costo ✓ Alta velocidad ✓ Amplia adopción ✗ Requiere cableado ✗ Interferencias (cobre)
IEEE 802.4 — Token Bus

Define redes tipo Token Bus. Implementa una red lógica en anillo con paso de testigo sobre cable coaxial. Solo el dispositivo que posee el token puede transmitir, evitando colisiones.

✓ Sin colisiones ✗ Mayor latencia ✗ Complejo ✗ En desuso
30 & 31 — Mensajería Electrónica

Protocolos de Correo

📤

SMTP

Simple Mail Transfer Protocol. Se usa para el envío de correos electrónicos entre servidores de correo. Puerto 25 / 587.

📥

POP3

Post Office Protocol v3. Para recepción de correos. Descarga los mensajes al cliente y los elimina del servidor. Puerto 110.

🔄

IMAP

Internet Message Access Protocol. Recepción con sincronización: los mensajes permanecen en el servidor accesibles desde múltiples dispositivos. Puerto 143.
